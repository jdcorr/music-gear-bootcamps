# M8 BOOTCAMP (14 DAYS)

Goal: become fast-fluent on the Dirtywave M8 (tracker workflow + sound design + sampling + effects/mix + rendering).

## END-OF-BOOTCAMP MASTERY (what “expert” means)
1. Navigation confidence: you can move between key Views and edit without getting lost.
2. Musical output: you can make a drum groove, bass/lead, and texture/chords with intentional FX + mixing.
3. Sampling competence: you can create and edit a sampled instrument (trim/crop, silence, mono/downsample for performance).
4. Finishing competence: you can render/export your song in Render View.

## BOOTCAMP DAILY STRUCTURE (repeat every day)
- 10 minutes: navigation/shortcuts drill (Views + cursor movement + edit discipline)
- 40–60 minutes: build the day’s specific deliverable
- 10 minutes: “what changed & why” notes (keep it short)

## THE “KNOW THESE FIRST” SHORTCUTS / CONCEPTS
- SHIFT + DIRECTION: navigate between Views
- EDIT: enter/yes + start editing values
- OPTION: no/exit (varies by context)
- PLAY: starts/stops song from the cursor position
- Mini-map (bottom-right): where you are on the screen
- Learn view flow: Song View -> Chain View -> Phrase View -> Instrument View

## CORE SOUND ENGINES YOU’LL PRACTICE
- WAVSYNTH: wave-table + noise + filter modes + amplification/clipping behaviors
- MACROSYNTH: 44 models controlled primarily by TIMBRE and COLOR (plus DEGRADE/REDUX)
- FM SYNTH: 4-op FM with 12 algorithms (A/B/C/D operators, RATIO/LEV/FB, modulation slots)
- SAMPLER: plays .wav, has Sample Editor for recording/editing and optimization

## SPECIAL PERFORMANCE/QUALITY CONCEPT (important for “expert” results)
- If you see “CPU TOO BUSY” during playback, it’s often SD streaming / sample bandwidth.
  - Actions: convert stereo to mono, reduce bit depth and/or sample rate, avoid playing samples too high above recorded pitch.

## WEEK-LIKE LEARNING PATH (what you’ll do)
- Days 1–3: UI + sequencing workflow + build instruments (Wavsynth first)
- Days 4–5: Macrosynth + FM Synth sound design competence
- Days 6–7: sampling + sample editor editing
- Days 8–11: put together a real arrangement + variations
- Days 12–14: render + quality pass + troubleshooting

---

## DAY 1: UI SURVIVAL + SONG STRUCTURE
### Task
- Load a demo song
- Play it from different cursor positions
- Navigate these Views repeatedly until it feels automatic:
  - Song View
  - Chain View
  - Phrase View
  - Instrument View
- Create a tiny phrase on ONE track (4–8 steps)
- Edit at least one value using EDIT (example: note, velocity, or instrument assignment)

### Deliverable
- One phrase that loops cleanly, and you can reliably start/stop from different cursor positions.

---

## DAY 2: SEQUENCER CRAFT (GROOVE + CHAINS/PHRASES)
### Task
- Build a 1–2 bar drum groove on Track 1
- Make at least 2 phrases on the drum track:
  - Phrase A: simpler pattern (core groove)
  - Phrase B: variation (add/remove/ratchet)
- Use chain/phrase structure so you don’t “rebuild everything manually”

### Deliverable
- “A then B then A” drum structure that feels musical (not random).

---

## DAY 3: WAVSYNTH MASTERY (FAST SOUND DESIGN)
### Task
- Create 1 Wavsynth instrument and dial it into a usable role
- Learn and commit to:
  - SHAPE (start simple: sine/saw/triangle)
  - FILTER type + CUTOFF + RES
  - AMP behavior (choose a limiter/clipping style so it sits in the mix)
- Make 3 variations of the same idea:
  - clean
  - warm
  - aggressive
- Use each variation in different sections (intro/build/drop)

### Deliverable
- 3 Wavsynth instruments + a short song section that uses them intentionally.

---

## DAY 4: MACROSYNTH MASTERY (MUSICAL PARAMETER THINKING)
### Task
- Pick one Macrosynth SHAPE model
- Learn how TIMBRE and COLOR move the sound (only those first)
- Then add DEGRADE/REDUX sparingly for texture
- Create:
  - 1 bass-like patch
  - 1 lead/texture patch

### Deliverable
- 2 Macrosynth instruments that sound good immediately on their own.

---

## DAY 5: FM SYNTH MASTERY (CONTROLLED HARMONICS)
### Task
- Create an FM Synth instrument
- Pick one ALGO
- Set stable starting points:
  - Keep operator levels modest at first
  - Keep feedback (FBK) controlled (don’t max instantly)
- Use RATIO on at least one operator for octave-ish motion
- Make:
  - a plucky tone
  - a bell-ish / metal-ish tone

### Deliverable
- 2 FM instruments with clearly different timbre goals.

---

## DAY 6: SAMPLER BOOT (LOAD/RECORD + FIRST INSTRUMENT)
### Task
- Create a Sampler instrument
- Load a WAV sample (or record in Sample Editor, depending on your setup)
- Learn the Sample Editor workflow:
  - select range
  - process (at least one operation)
  - save and confirm it plays correctly
- Ensure it plays melodically (use slicing/loop strategy appropriate to your sample)

### Deliverable
- One sampled instrument that you can play as notes and use in the song.

---

## DAY 7: SAMPLE EDITOR POWER-UP (PRODUCER EDIT SKILLS)
### Task
- Edit your sample instrument using at least 3 tools:
  - CROP
  - NORMALIZE
  - SILENCE
  - REVERSE (optional)
  - MONO conversion (stereo -> mono)
  - DOWNSAMPLE / reduce bit depth (for performance)
- If you hit performance limits (“CPU TOO BUSY”):
  - convert to mono when stereo isn’t necessary
  - reduce bit depth / sample rate
  - avoid playing too far above recorded pitch range

### Deliverable
- A “clean + efficient” sample instrument.

---

## DAY 8: EFFECTS PIPELINE + MIX DISCIPLINE
### Task
- Navigate and use the dedicated mix/effects pipeline views:
  - Mixer View (including output behavior)
  - EQ Editor View
  - Limiter & Mix Scope View
  - Effect Settings View
- Make mix decisions like a producer:
  - EQ: reduce clutter first, then boost only if needed
  - Limiter: tame peaks without making it harsh

### Deliverable
- Your track section sounds “released”: not too quiet, not harsh, not muddy.

---

## DAY 9: PUT IT TOGETHER (3-TRACK MINI SONG)
### Task
- Build a coherent mini-song (8–16 bars)
- Use at least:
  - drums (tracker sequencing)
  - bass or lead (Wavsynth/Macrosynth/FM)
  - texture/chords (another engine or your sampler)
- Create at least 2 sections using chains/phrases.

### Deliverable
- A full mini-song (it can be short, but it must have structure).

---

## DAY 10: VARIATION / EVOLUTION SPRINT
### Task
- Add one type of change per instrument (choose one per track):
  - timbre motion (filter/waveshape behavior)
  - distortion/clipping behavior changes
  - note density / phrasing changes
- Keep changes purposeful: intro -> build -> drop.

### Deliverable
- Version 2 of your mini-song with real evolution.

---

## DAY 11: EXPAND ARRANGEMENT
### Task
- Extend arrangement toward 24–48 bars
- Use phrases/chains to reduce repetitive copy/paste fatigue

### Deliverable
- Recognizable sections and a satisfying ending.

---

## DAY 12: EXTERNAL INTEGRATION (OPTIONAL)
Choose one:
- If you use external MIDI gear: record/control one meaningful part.
- If not: skip and improve internal mix/instruments.

### Deliverable
- Either external cameo OR upgraded internal quality.

---

## DAY 13: RENDER DISCIPLINE (EXPORT SOMETHING “FINAL”)
### Task
- Use Render View to render your song
- Make at least:
  - one mixed/final render

### Deliverable
- A rendered WAV you’d actually share.

---

## DAY 14: QUALITY PASS + TROUBLESHOOTING
### Task
- Stress test:
  - Does playback hold for the full length?
  - Any “CPU TOO BUSY” risk?
  - Sample streaming issues?
- Rework the weakest phrase (only one, but make it count)
- Final EQ/limiter tweak pass

### Deliverable
- Final project + short checklist:
  - what instruments you used
  - what editing/optimization you did (mono/downsample etc.)
  - what made it sound better
